# v5.4 核心循环、Boss 与 UI 设计

## 目标

让 Skyline Legion 更接近 TopWar 类买量小游戏的 30 秒展示节奏：开局弱、成长快、选门必须移动、Boss 战短且有明确躲避压力。

## 设计

- 开局从 1 个小兵开始，普通弹道也从 1 条开始，随着小兵数量增长逐步增加弹道。
- 倍增门不再显眼撞到小数字上限。门的显示值允许继续被充高，真正控量依靠初始人数、负门数值、敌群损耗和后台人数保护。
- 门区充能按玩家所在半区限制。玩家在右侧时，主要只有右门能被充能；玩家在左侧时，主要只有左门能被充能，避免后期火力同时把两边门打满。
- 后期负门初始值扩大到大约 `-99 ~ -30`。玩家需要主动选边，不能靠站中间吃火力覆盖。
- 暂停界面增加“重新开始”，方便试玩时快速重开。
- Boss 战目标时长压到约 10 秒。Boss 普通炮弹继续保持单发，但落点范围更大，且范围内所有小兵都会死亡，形成明确躲避压力。
- UI 优化以实用反馈优先：HUD 轻微玻璃质感、数值/按钮动态、暂停面板更完整，避免大改风格导致试玩节奏变乱。

## 验证标准

- `npm run build` 通过。
- 自动试玩能完整进入并结束 Boss 战，记录门选择、峰值小兵、Boss 起始人数、Boss 时长和最终人数。
- Boss 专项验证要记录 `boss-fire` 仍为单发，且 `boss-projectile-hit` 可以造成多名小兵损失。
- 暂停界面必须能看到继续和重新开始两个操作。
