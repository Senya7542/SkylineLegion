# v5.5 门阻挡、门分布与 Boss 索敌设计

## 目标

修正 v5.4 的门充能改动：门应该阻挡当前区域子弹，站中间同时刷两个门是顺畅且符合预期的玩法。继续保留后期负门压力，并缩小 Boss 炮弹范围，让它锁定最前方小兵而不是飞机位置。

## 设计

- 移除“玩家必须明显偏左/偏右才允许给对应侧门充能”的限制。
- 当前区域未结算的门继续作为子弹阻挡物；子弹命中门后停止并给命中的那一侧充能。
- 门的正负分布不再固定为左红右蓝。每组门按种子随机决定：
  - 常规组：一侧奖励，一侧惩罚。
  - 最后一组：允许两侧都是负数红门，左/右只区分惩罚大小。
- Boss 炮弹杀伤半径缩小到 v5.4 的约一半。
- Boss 炮弹目标改为最前方小兵的世界位置，找不到小兵时才回退到飞机位置。
- 保持 Boss 战短节奏，不新增特色攻击。

## 验证标准

- `npm run build` 通过。
- 自动试玩能验证门两侧都有机会出现红/蓝或双红负门。
- 调试日志能记录 Boss 炮弹半径和群伤损失。
- 浏览器无运行时错误。
