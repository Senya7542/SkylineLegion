# v5.6 重开、Boss 索敌与门数值设计

## 目标

修正 v5.5 后的最新反馈：Boss 炮弹应锁定距离 Boss 最近的小兵范围；第一次开始游戏时门左右也要随机；第二区域门数值差距收窄；暂停、继续、重开后不能出现子弹穿过门或敌人的状态错位。

## 设计

- Boss 炮弹目标从最大 `unit.z` 改为最小 `unit.z`，也就是当前坐标里更靠近 Boss 的前排小兵。
- 每次开始游戏都生成新的 `runSeed`，包括第一次从菜单进入游戏。
- 重新开始时清理旧的 intro timer，并用新的 seed 强制 remount `World`，避免暂停/继续状态下残留旧世界。
- 门数值整体降档：第二区域接近 `-7 / +15` 的尺度，后续区域只适度拉开差距，最后保留双负门但不再过分极端。
- 保留站中间可同时刷两个门；当前区域门继续阻挡子弹。
- Boss 战控制在短展示节奏内，炮弹仍是群体伤害，但杀伤半径缩小到更容易读懂和躲避的范围。

## 验证标准

- `npm run build` 通过。
- 第一次开始和重新开始都能得到不同门布局。
- 暂停 -> 继续 -> 重新开始后，子弹仍能命中门和敌人。
- Boss `boss-fire` 日志记录 `front-troop` 来源和目标 `z`。
- 自动完整局可以进入 Boss，并在约 10 秒内结束 Boss 战。
