# v5.3 门、敌群与 Boss 炮弹平衡设计

## 目标

解决后半段部队数量过早撞到上限、最后两组门看起来不再增长、Boss 双发炮弹压迫方式不够清晰的问题。

## 设计

- 保留真实部队上限 320，避免极端倍增导致性能和结算失控。
- 将可见部队上限从 96 提到 128，让增长反馈更明显；当真实人数超过可见上限时，HUD 显示可见上限提示。
- 后两个区域的负数门初始值加深，第四区最重；正数门保留成长空间，但初始收益更克制。
- 第三、第四波敌人数量增加，让区域内损耗成为主要控量手段，而不是单纯依靠硬上限。
- Boss 普通炮弹由一轮双发改为一轮单发；单发炮弹的命中范围加大，让玩家更容易读懂威胁，但每轮不会同时覆盖两侧。

## 验证标准

- `npm run build` 必须通过。
- 自动试玩要能进入 Boss，并记录门收益、敌群损耗、Boss 起始人数和最终结果。
- 不能出现运行时错误；已有 Three.js 精度或 Clock 警告只记录，不作为失败。
